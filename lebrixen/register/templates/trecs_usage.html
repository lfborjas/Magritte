{%trans "Usage"%}

{%blocktrans%}You can use T-recs in any kind of application, from a desktop text processor to a full web course management system. The easiest way is to use the javascript widget, but, if it doesn't meet your needs, you can always use the REST API{%endblocktrans%}

{%trans "User registration"%}

{%blocktrans%}You can use the dashboard or implement your own user management with the API calls for User administration{%endblocktrans%}

Widget

  • {%trans "Include the file http://trecs.com/static/js/service/recommender.js"%}
  • {%trans "Include a script with the configuration"%}

API

{%blocktrans%}The API always returns a JSON text with the same structure: only the data property depends on the call{%endblocktrans%}

{%trans "Get Recommendations call"%}

http://trecs.com/api/getRecommendations?appId={%trans "your-app-id"%}&appUser={%trans "registered-user"%} &context={%trans "free-text-to-analyze"%} &lang={%trans "content-language"%} [&callback={%trans "(for jsonp calls)"%}]

{%trans "Update profile call"%}

http://trecs.com/api/getRecommendations?appId={%trans "your-app-id"%}&appUser={%trans "registered-user"%} &context={%trans "free-text-to-analyze"%} &lang={%trans "content-language"%} &docs={%trans "feedback-documents"%} [&callback={%trans "(for jsonp calls)"%}]

{%trans "User administration"%}

{%blocktrans%}All of these calls require basic http authentication: the username is the name that you registered as your application name/url and the password is the one sent to you when you registered {%endblocktrans%}

http://trecs.com/api/registerUser?appId={%trans "your-app-id"%}&user={%trans "user-to-register"%} [&callback={%trans "(for jsonp calls)"%}] http://trecs.com/api/getUsers?appId={%trans "your-app-id"%}[&callback={%trans "(for jsonp calls)"%}] http://trecs.com/api/deleteUser?appId={%trans "your-app-id"%}&user={%trans "user-to-delete"%} [&callback={%trans "(for jsonp calls)"%}]
